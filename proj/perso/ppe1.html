PPE 1 | Gestion de flotte web

PROJET

Le projet est conçu pour qu'une entreprise de location de véhicules, possédant plusieurs agences de véhicules, puisse gérer les différents facteurs pour la gestion de location.


L'entreprise pourra également gérer la disponibilité des véhicules pour les différentes commandes des clients.


Il y aura plusieurs rôles pour les utilisateurs de l'entreprise comme la RH ou le chef d'agences qui auront un rôle bien précis sur le site.


Pour ce projet, nous partirons sur une API Laravel que nous pourrons réutiliser pour une application mobile future, et ainsi liée cette API au Front codé en PHP.


L'API Laravel consistera à créer toutes les fonctionnalités nécessaires pour l'entreprise, notamment la gestion des agences, des véhicules et des commandes.

API Laravel

Tout d'abord, nous commençons un dossier pour l'api laravel dans laquelle nous disposerons les fonctionnalités de notre application et les chemins de ces actions appelés routes.


Nous allons créer différents Controllers que nous rendons disponibles via une route. Ces routes seront ensuite appelées sur le Front pour utiliser les différentes fonctionnalités.


Il y a donc plusieurs Controllers pour pouvoir gérer les véhicules, les fournisseurs, les agences et les commandes.


A savoir, pour la connexion, l'utilisateur se vera recevoir un mail comportant son mot de passe pour pouvoir se connecter.


En terme de fonctionnalités, l'entreprise pourra créer, modifier ou supprimer des véhicules, des fournisseurs, des agences et des commandes selon le rôle de l'utilisateur.


Le fichier web.php nous permet de lié les routes de l'API au Front grâce à des requêtes.





Front







Après avoir créer les fonctionnalités avec l'api laravel, nous devons appeller les routes de l'api via des requêtes.


Au moment de récupérer les fonctionnalités de l'API, nous devons faire attention à bien afficher les pages en fonction du rôle de l'utilisateur.
